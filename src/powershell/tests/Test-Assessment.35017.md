Without default labels, users must actively choose a label for every item they create, increasing friction and inconsistent labeling. Default labels provide a baseline classification that can be overridden when necessary, reducing decision fatigue and ensuring content has at least a minimum level of classification.

Without default labels, unclassified content might bypass data loss prevention (DLP) policies that rely on label detection. Different workloads should each have appropriate default labels configured.

- [Publish sensitivity labels by creating a label policy](https://learn.microsoft.com/purview/create-sensitivity-labels?wt.mc_id=zerotrustrecommendations_automation_content_cnl_csasci#publish-sensitivity-labels-by-creating-a-label-policy)
- [Configure a default sensitivity label for a SharePoint document library](https://learn.microsoft.com/purview/sensitivity-labels-sharepoint-default-label?wt.mc_id=zerotrustrecommendations_automation_content_cnl_csasci)
