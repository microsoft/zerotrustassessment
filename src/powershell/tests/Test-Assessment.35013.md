Without encryption, sensitivity labels only classify content as sensitive without preventing unauthorized access. Encryption-enabled labels apply Azure Rights Management protection that enforces access controls, ensuring protection persists regardless of where content is stored or shared. For example, users can still forward "Confidential" documents to unauthorized recipients unless encryption restricts file access based on identity.

Organizations using labels without encryption gain visibility but lack technical enforcement. Encrypted labels ensure only authorized users can decrypt content, preventing data exfiltration even if files are leaked or improperly shared. At least one encryption-enabled label should exist for high-value data requiring protection beyond classification.

**Remediation action**

- [Restrict access to content by using encryption in sensitivity labels](https://learn.microsoft.com/purview/encryption-sensitivity-labels?wt.mc_id=zerotrustrecommendations_automation_content_cnl_csasci)
