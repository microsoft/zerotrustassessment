Without App Control for Business (formerly Windows Defender Application Control, WDAC), every executable a user can drop onto disk — phishing-delivered payloads, malicious npm or PyPI dependencies, sideloaded utilities from a USB stick, ransomware droppers, living-off-the-land binaries that re-package legitimate Microsoft signed tools — can run with the user's full token, against corporate data and Entra ID resources, before any Endpoint Detection and Response (EDR) signal has a chance to fire. EDR is reactive: it sees and may quarantine *after* malware has executed, often after data has already been staged for exfiltration or after persistence has been established. App Control disrupts this kill chain at the kernel level by establishing a Windows code-integrity policy that the kernel itself enforces *before* any executable starts: only files signed by trusted publishers, on a trusted file path, or specifically allow-listed by the organisation are permitted to run; unsigned or unknown executables fail to launch even if SYSTEM-level malware tries to invoke them. Microsoft delivers App Control for Business via Intune's Endpoint Security → App Control for Business policy, which deploys a Windows Code Integrity (WDAC) policy to managed Windows endpoints. Without an enforced App Control policy assigned to the Windows fleet, ransomware and living-off-the-land binaries continue to have an unimpeded path to execution, and the entire Defender / compliance / Conditional Access pipeline downstream of execution is reduced to a detection-and-cleanup story rather than a prevention story.
